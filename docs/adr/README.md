# ADRs

This directory stores accepted architecture decision records that informed the current runtime foundation.

These ADRs are maintainer references, not the public support contract. Current support and compatibility promises live in the root README plus [../SUPPORT.md](../SUPPORT.md).

## Format

Each ADR must contain these sections:

- `Status`
- `Context`
- `Decision`
- `Consequences`
- `Non-Goals`
- `Rejected Alternatives`

## Rules

- Use ADRs for stable architectural decisions, not for aspirational feature lists.
- Keep historical architecture documentation separate from rewrite-target ADRs.
- When a decision changes, add a new ADR that supersedes the old one instead of silently rewriting history.
