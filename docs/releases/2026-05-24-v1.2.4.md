# v1.2.4 Release Notes

## Summary

- release tag: `v1.2.4`
- candidate commit SHA: commit tagged by `v1.2.4`
- release type: stable patch
- one-sentence user summary: Cursor package install support is reissued from current `main`, with a release guard that prevents branch-only stable releases.

## Public-Stable In This Release

- Cursor package manifests can install skills-first bundles into local Cursor plugin storage.
- The stable release asset workflow now fails before publishing if the selected tag is not current `origin/main`.

## Why This Release Matters

`v1.2.3` produced valid GitHub release binaries, but it was cut from a non-main branch. Downstream registry workflows key off the stable main release state, so this patch reissues the same user-facing Cursor support from the correct branch boundary and prevents repeating that failure mode.

## What Changed For Users

- No migration required.
- New installs use the latest GitHub release through the existing `curl | bash` installer path.
- Cursor users still reload Cursor or restart it after install, then open a new chat so plugin skills are loaded.

## External Smoke Status

- required: local Go module tests passed before tagging.
- integrationctl: local package tests passed.
- cli/plugin-kit-ai: local package tests passed.
- release-assets: planned through GitHub Actions for `v1.2.4`.
- downstream publishes: planned through explicit release workflow evidence after assets publish.

## Known Limitations

- Cursor still needs a native reload or restart after local plugin package materialization.
- npm publication depends on the repository npm publisher credentials or trusted publisher configuration matching the `plugin-kit-ai` and `plugin-kit-ai-runtime` package owners.

## Decision Record
