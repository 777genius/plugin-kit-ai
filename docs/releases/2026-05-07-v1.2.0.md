# v1.2.0 Release Notes

## Summary

- release tag: `v1.2.0`
- candidate commit SHA: commit tagged by `v1.2.0`
- release type: stable minor
- one-sentence user summary: `plugin-kit-ai skills` can now manage external Agent Skills packages through the upstream npm `skills` CLI, and integration lifecycle output is easier to read.

## Public-Stable In This Release

- `plugin-kit-ai skills install|list|update|remove` as an npm-backed wrapper around `skills@1.5.5`.
- No Go SDK or `plugin-kit-ai-runtime` API promotion in this release.

## Why This Release Matters

Teams that reuse Agent Skills across Codex, Claude Code, Gemini CLI, and other agents can now use `plugin-kit-ai` as the entrypoint instead of dropping down to `npx skills` directly. Integration lifecycle output also now reports planned and applied target state in user-facing language rather than raw internal fields.

## What Changed For Users

- Added `plugin-kit-ai skills install <source>` with alias `add`.
- Added `plugin-kit-ai skills list` with alias `ls`.
- Added `plugin-kit-ai skills update` with alias `upgrade`.
- Added `plugin-kit-ai skills remove` with alias `rm`.
- Pinned the default external skills CLI to `skills@1.5.5`, with `--skills-cli-version` as an override.
- Preserved existing repo-local `skills init|validate|generate` authoring behavior.
- Improved integration add/update/remove reports with readable summaries, target status groups, activation notes, and follow-up steps.

## What To Do Now

- Upgrade to `v1.2.0` when you want `plugin-kit-ai` to install or list external Agent Skills packages.
- Example: `plugin-kit-ai skills install flutter/skills --global --all`.
- No migration is required for existing `plugin-kit-ai skills init|validate|generate` projects.

## Still Public-Beta

- The external skills wrapper follows the upstream `skills` CLI behavior and inherits its supported agent matrix.
- Existing integration lifecycle management remains governed by the install/update control-plane beta contract.

## Beta Contract Changes

- External Agent Skills management is added as a new CLI surface and stays bounded to the upstream `skills@1.5.5` command contract.
- Integration report text changed from raw target fields to human-readable status lines.
