## What Went Wrong

- This release corrected the public import path and added dual SDK tagging, but it did not move the physical module root to `sdk/`.
- As a result, `github.com/777genius/plugin-kit-ai/sdk@v1.0.3` does not resolve as a normal Go module release.
- Use `v1.0.4` or newer for Go SDK module consumption.
