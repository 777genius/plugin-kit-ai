# v1.2.3 Release Notes

## Summary

- release tag: `v1.2.3`
- candidate commit SHA: commit tagged by `v1.2.3`
- release type: stable patch
- one-sentence user summary: Cursor can now install skills-first plugin bundles instead of being limited to MCP config projection.

## Public-Stable In This Release

- Cursor package manifests generated by `plugin-kit-ai generate --platform cursor` can reference generated `skills` through `./skills/`.
- `plugin-kit-ai add` materializes Cursor plugin bundles into `~/.cursor/plugins/local/<plugin>`.
- Existing Cursor MCP-only integrations continue to use `.cursor/mcp.json`.

## Why This Release Matters

Skills-first plugins such as `agent-code-navigator` intentionally avoid generic cross-agent MCP wiring. Before this release, the Cursor target in the installer was MCP-centric, so those packages could validate and publish Cursor manifests but could not be installed through the one-command path.

## What Changed For Users

- No migration required for existing Cursor MCP integrations.
- Cursor users can install packages that ship `.cursor-plugin/plugin.json` plus `skills/`.
- After install, reload Cursor or restart it, then open a new chat so plugin skills are loaded.

## Known Limitations

- Cursor still needs a native reload or restart after local plugin package materialization.
- Generic Serena/CGC MCP auto-wiring remains intentionally out of scope for skills-first packages.
