# Code Of Conduct

## Our Standard

This project is committed to a respectful, practical, and high-signal community.
We want discussion to stay useful, technically honest, and safe for people with
different levels of experience.

Examples of behavior that helps:

- assume good intent and ask for clarification before escalating
- keep feedback concrete, testable, and tied to the code or docs
- share context when disagreeing, especially around security or compatibility
- help newcomers understand the contract and boundaries of the project

Examples of behavior that is not acceptable:

- harassment, insults, dogpiling, or personal attacks
- bad-faith review, trolling, or deliberately wasting maintainer time
- publishing private or security-sensitive information without consent
- hostile gatekeeping or dismissing contributors for asking basic questions

## Scope

This applies to repository discussions, issues, pull requests, release notes,
and any other official project spaces.

## Enforcement

Project maintainers may remove, edit, or reject content that breaks this code
of conduct. Repeated or severe violations may lead to temporary or permanent
loss of participation in project spaces.

## Reporting

For conduct issues, contact the maintainers privately. For undisclosed security
issues, use the process in [SECURITY.md](./SECURITY.md) instead of a public
issue.
