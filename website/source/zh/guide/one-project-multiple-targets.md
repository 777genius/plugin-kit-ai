# 一个项目，多个目标

在第一个可用的存储库之后使用此页面，此时真正的问题是：同一个存储库是否应该增长，如果是，增长到什么程度？

## 简短规则

当相同的插件逻辑、发布意图和所有权模型仍然保持在一起时，一个存储库可以安全地覆盖多个输出。

## 一个仓库何时应该扩展

在以下情况下增长相同的存储库：

- 插件行为仍然是一个连贯的产品
- 新的输出是提供相同插件的另一种方式
- 一个团队仍然可以干净地拥有所创作的源代码
- 重新生成和验证仍然使存储库易于审查

## 何时一个仓库应该保持狭窄

当当前输出已经解决了实际需求并且额外的输出只会增加维护开销时，请保持仓库聚焦。

## 何时拆分仓库

当产品在实践中不再是一件事时，拆分仓库：

- 不同的团队拥有工作
- 发布时间不同
- 行为超出了简单的目标适应范围
- 与两个更小的仓库相比，一个大仓库会更难推理

## 安全心理模型

开始缩小范围，验证一个工作输出，然后才使用另一个受支持的输出来扩展存储库。
