# v1.1.2

发布日期：`2026-04-19`

## 为什么这个版本重要

`v1.1.2` 修复了完整 multi-target add flow 中的 Gemini 部分，当 source 来自 GitHub repo path 或解析到该路径的 first-party alias 时尤其如此。

## 对用户有什么变化

- GitHub repo-path sources 的 Gemini 安装现在会走 managed local projection flow
- `gitlab`、`notion` 之类的 first-party aliases 不会再因为 Gemini lane 而破坏完整的 `add` run
- 受支持 targets 之间的 managed multi-agent install story 再次保持一致

## 这在实践中意味着什么

- `plugin-kit-ai add gitlab` 和 `plugin-kit-ai add notion` 现在会在当前 patch 线上重新完成
- 团队可以继续使用简单的 first-party alias 路径，而不必绕开 Gemini 特有的 repo-path 行为
- 当前 public baseline 现在同时覆盖了 authored-layout compatibility 和 Gemini repo-path fix

## v1.1.2 之后的推荐默认值

- 如果目录已经提供 first-party aliases，就继续把它们作为正常的公共安装路径
- 让 managed repo-path installs 保持在当前 patch 线或更新版本
- 如果 Gemini 之前失败过，请在 `v1.1.2` 或更新版本上重新运行完整 multi-target add flow

## 现在该做什么

- 如果 multi-target installs 在 Gemini 步骤失败，请升级到 `v1.1.2` 或更高版本
- 为受影响的 first-party aliases 重新运行 add flow
- 将 `v1.1.2` 视为当前安装指南和示例的公开基线
