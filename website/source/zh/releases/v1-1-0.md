# v1.1.0

发布日期：`2026-04-18`

## 为什么此版本很重要

`v1.1.0` 是一个面向将存储库视为代理集成状态所有者的团队的加固版本。它不会改变默认推荐路径，但会让受支持的多代理设置中的添加、更新和删除流程更值得信赖。

## 用户发生了什么变化

- repo-owned 集成生命周期流程现在在 Claude、Codex、Cursor 和 OpenCode 的托管路径上具有更强的端到端验证
- 公共发布面现在在 docs、starter 和 helper package pin 中统一指向 `v1.1.0` 基线
- runtime-package 升级 smoke 现在会检查升级到当前公共包版本，而不是停留在更早的补丁线

## 这在实践中意味着什么

- 如果您的存储库为多个目标管理代理配置，则添加、更新和删除操作现在会在发布前以更真实的方式被演练
- 如果您的团队依赖公共 docs 或 starter README，可见的版本 pin 现在与当前发布基线一致，而不是混用旧值
- 如果您发布或交接基于 starter 的存储库，发布故事会更容易解释，因为基线、docs 和 package pin 现在已经对齐

## v1.1.0 之后的推荐默认值

- 当您想要最强的通用生产路径时，继续将 Go 作为默认值
- 当这些 repo-local 解释型路径确实是团队的有意识选择时，再使用 Node 或 Python shared runtime-package 路径
- 当存储库拥有代理设置时，将 repo-owned managed integrations 视为经过验证的生命周期

## 现在该做什么

- 更新任何仍然提到 `v1.0.6` 的复制 docs 或 starter 片段
- 如果您依赖跨代理的 repo-owned integration setup，请用 `v1.1.0` 重新运行常规的 add、update 和 remove 工作流
- 如果您从头开始，请使用当前 starter pin 和此发行说明作为基线
