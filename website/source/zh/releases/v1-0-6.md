# v1.0.6

发布日期：`2026-03-30`

## 为什么此版本很重要

在 `v1.0.6` 版本中，共享运行时包成为 Python 和 Node 团队的明确公共路径，而不是半支持的工作流程。

## 用户发生了什么变化

- `plugin-kit-ai init` 现在可以围绕 `--runtime-package` 搭建 Python 和 Node 项目
- 共享 `plugin-kit-ai-runtime` 路径现在是已批准的 Python 和 Node 选项的受支持路径
- 更新了入门模板和公共示例以匹配当前推荐的设置
- 加强了版本验证和版本同步检查，以便发布的版本与记录的安装路径保持一致

## 这在实践中意味着什么

- 如果您的团队想要主要的非Go路径，Node/TypeScript加上`--runtime-package`现在是推荐的选择
- 如果您的团队是 Python-first 并保持存储库本地，则共享运行时包现在是首选受支持路径
- `doctor` 和运行时要求消息传递更清晰，因此在指责项目之前更容易捕获环境故障

## v1.0.6 之后的推荐默认值

- 当您想要最强的生产路径和最小的下游引导摩擦时，选择 Go
- 当您想要主要的非 Go 支持路径时，选择 Node/TypeScript 和 `--runtime-package`
- 当项目是 repo-local 并且团队有意 Python-first 时，选择 Python 和 `--runtime-package`

## 现在该做什么

- 如果您已经使用 Python 或 Node，请重新检查 `--runtime-package` 现在是否是您的存储库的更好默认值
- 如果您仍然依赖供应的帮助程序文件作为长期默认设置，请将该设置与当前的 `plugin-kit-ai-runtime` 指南进行比较
- 如果您是新手，上面的默认设置是最简单的起点

将此版本与[安装](/zh/guide/installation)、[选择运行时](/zh/concepts/choosing-runtime) 和[安装通道](/zh/reference/install-channels) 配对。
