# v1.0.6

Дата релиза: `2026-03-30`

## Почему этот релиз важен

`v1.0.6` - это релиз, в котором shared runtime package стал понятным публичным путём для Python и Node команд, а не частичным или оговорочным workflow.

## Что изменилось для пользователей

- `plugin-kit-ai init` теперь умеет scaffold'ить Python и Node проекты вокруг `--runtime-package`
- общий путь через `plugin-kit-ai-runtime` теперь рассматривается как поддерживаемый путь для одобренных Python и Node вариантов
- starter templates и публичные примеры обновлены под текущую рекомендуемую настройку
- release verification и version-sync checks стали жёстче, чтобы опубликованный релиз не расходился с задокументированным install path

## Что это значит на практике

- если команде нужен основной non-Go путь, Node/TypeScript плюс `--runtime-package` теперь является рекомендуемым выбором
- если команда Python-first и проект остаётся repo-local, shared runtime package теперь является предпочтительным поддерживаемым путём
- `doctor` и сообщения про runtime requirements стали понятнее, поэтому проблемы окружения легче поймать до того, как обвинять сам проект

## Рекомендуемые default choices после v1.0.6

- выбирайте Go, когда нужен самый сильный production path и минимум downstream bootstrap friction
- выбирайте Node/TypeScript с `--runtime-package`, когда нужен основной поддерживаемый non-Go путь
- выбирайте Python с `--runtime-package`, когда проект repo-local и команда осознанно Python-first

## Что делать теперь

- если вы уже используете Python или Node, перепроверьте, не стал ли `--runtime-package` лучшим выбором для вашего repo
- если вы до сих пор считаете vendored helper files основным долгосрочным путём, сравните этот вариант с текущими рекомендациями по `plugin-kit-ai-runtime`
- если вы стартуете с нуля, начинайте с recommended defaults выше
