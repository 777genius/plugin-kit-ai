# v1.1.0

Дата релиза: `2026-04-18`

## Почему этот релиз важен

`v1.1.0` - это hardening-релиз для команд, где repo действительно владеет состоянием agent integrations. Он не меняет основной recommended path, но делает add, update и remove заметно более доверяемыми в поддерживаемой multi-agent настройке.

## Что изменилось для пользователей

- lifecycle repo-owned integrations теперь жёстче проверяется end-to-end для managed путей Claude, Codex, Cursor и OpenCode
- публичная release-витрина теперь указывает на единый baseline `v1.1.0` в docs, starters и helper-package pins
- smoke-проверки runtime packages теперь проверяют апгрейд до текущей публичной версии пакета, а не останавливаются на более старой patch-линии

## Что это значит на практике

- если repo управляет agent config сразу для нескольких target'ов, операции add, update и remove теперь реалистичнее прогоняются до релиза
- если команда ориентируется на публичные docs или starter README, видимые version pins теперь совпадают с текущим опубликованным baseline вместо смеси старых значений
- если вы публикуете starter-based repo или передаёте его дальше, release story стало проще объяснять, потому что baseline, docs и package pins теперь согласованы

## Рекомендуемые defaults после v1.1.0

- сохраняйте Go как default, когда нужен самый сильный общий production path
- используйте Node или Python shared runtime-package path только когда эти repo-local interpreted lanes действительно осознанны для команды
- относитесь к repo-owned managed integrations как к проверенному lifecycle, если repo владеет agent setup

## Что делать теперь

- обновите локально скопированные docs или starter snippets, если там ещё фигурирует `v1.0.6`
- если вы полагаетесь на repo-owned integration setup у нескольких агентов, перепроверьте ваш обычный add, update и remove workflow уже на `v1.1.0`
- если стартуете с нуля, используйте текущие starter pins и этот release note как baseline
