# v1.1.1

Дата релиза: `2026-04-19`

## Почему этот релиз важен

`v1.1.1` возвращает installer compatibility в случаях, когда source repo всё ещё использует либо legacy-дерево `src/plugin.yaml`, либо текущее дерево `plugin/plugin.yaml`.

## Что изменилось для пользователей

- integration install detection теперь принимает оба authored layout во время add flows
- first-party aliases вроде `gitlab` и `notion` больше не падают только из-за того, что upstream repo всё ещё на старом дереве
- текущий layout через `plugin/` остаётся рекомендуемым путём для новых repo

## Что это значит на практике

- `plugin-kit-ai add <source>` снова работает на текущем first-party catalog
- команде больше не нужно заранее знать, какой authored layout до сих пор живёт в source repo
- публичная patch-линейка CLI снова соответствует совместимости, которую обещают опубликованные install examples

## Рекомендуемые defaults после v1.1.1

- держите `plugin/plugin.yaml` как canonical layout для новых repo
- воспринимайте `src/plugin.yaml` как legacy compatibility input, а не как preferred new path
- перепроверьте неудавшиеся first-party alias installs уже на этом патче или новее

## Что делать сейчас

- обновитесь до `v1.1.1` или новее, если на опубликованном CLI падали `add gitlab` или `add notion`
- повторно запустите add flow, если предыдущая установка оборвалась на середине
- оставляйте новые repo на стандартном authoring layout через `plugin/`
