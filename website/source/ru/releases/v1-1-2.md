# v1.1.2

Дата релиза: `2026-04-19`

## Почему этот релиз важен

`v1.1.2` чинит Gemini-часть полного multi-target add flow, когда source приходит из GitHub repo path или из first-party alias, который на него резолвится.

## Что изменилось для пользователей

- Gemini installs для GitHub repo-path sources теперь идут через managed local projection flow
- first-party aliases вроде `gitlab` и `notion` больше не ломают полный `add` run из-за Gemini lane
- managed multi-agent install story снова ведёт себя согласованно на поддерживаемых targets

## Что это значит на практике

- `plugin-kit-ai add gitlab` и `plugin-kit-ai add notion` снова доходят до конца на текущей patch-линейке
- команды могут оставаться на простом пути через first-party aliases без Gemini-specific workaround
- текущий public baseline теперь закрывает и authored-layout compatibility, и Gemini repo-path fix

## Рекомендуемые defaults после v1.1.2

- используйте first-party aliases как normal public install path, если каталог уже их даёт
- держите managed repo-path installs на текущем патче или новее
- если Gemini раньше падал, перепроверьте full multi-target add flow уже на `v1.1.2` или новее

## Что делать сейчас

- обновитесь до `v1.1.2` или новее, если full multi-target install падал на Gemini step
- заново запустите add flow для затронутых first-party aliases
- считайте `v1.1.2` текущим public baseline для install guidance и примеров
