# v1.0.6

Date de sortie : `2026-03-30`

## Pourquoi cette version est importante

`v1.0.6` est la version dans laquelle les packages d'exécution partagés sont devenus le chemin public clair pour les équipes Python et Node au lieu d'un flux de travail à moitié pris en charge.

## Ce qui a changé pour les utilisateurs

- `plugin-kit-ai init` peut désormais échafauder les projets Python et Node autour de `--runtime-package`
- le chemin partagé `plugin-kit-ai-runtime` est désormais un chemin pris en charge pour les options approuvées Python et Node
- les modèles de démarrage et les exemples publics ont été actualisés pour correspondre à la configuration actuelle recommandée
- La vérification des versions et les contrôles de synchronisation des versions ont été renforcés afin que la version publiée reste alignée sur le chemin d'installation documenté.

## Ce que cela signifie en pratique

- si votre équipe souhaite le chemin principal non-Go, Node/TypeScript plus `--runtime-package` est désormais le choix recommandé
- si votre équipe est Python-first et reste locale au dépôt, le package d'exécution partagé est désormais le chemin pris en charge préféré
- `doctor` et les messages sur les exigences d'exécution sont plus clairs, de sorte que les défaillances de l'environnement sont plus faciles à détecter avant de blâmer le projet

## Valeurs par défaut recommandées après v1.0.6

- choisissez Go lorsque vous souhaitez le chemin de production le plus solide et le moins de frictions d'amorçage en aval
- choisissez Node/TypeScript avec `--runtime-package` lorsque vous voulez le chemin principal non pris en charge par Go
- choisir Python avec `--runtime-package` lorsque le projet est repo-local et que l'équipe est intentionnellement Python-first

## Que faire maintenant

- si vous utilisez déjà Python ou Node, vérifiez à nouveau si `--runtime-package` est désormais la meilleure valeur par défaut pour votre dépôt
- si vous comptez toujours sur les fichiers d'aide du fournisseur comme valeur par défaut à long terme, comparez cette configuration aux conseils `plugin-kit-ai-runtime` actuels
- si vous recommencez à zéro, les valeurs par défaut ci-dessus sont le point de départ le plus simple
