# v1.1.2

Date de sortie : `2026-04-19`

## Pourquoi cette version compte

`v1.1.2` corrige la partie Gemini des flux d'ajout multi-target complets lorsque la source vient d'un GitHub repo path ou d'un alias first-party qui s'y résout.

## Ce qui a changé pour les utilisateurs

- les installations Gemini pour les sources GitHub repo-path passent désormais par le flux géré de projection locale
- les aliases first-party comme `gitlab` et `notion` ne cassent plus les exécutions complètes de `add` à cause de la branche Gemini
- l'histoire d'installation multi-agent gérée reste de nouveau alignée entre les targets pris en charge

## Ce que cela signifie en pratique

- `plugin-kit-ai add gitlab` et `plugin-kit-ai add notion` se terminent de nouveau sur la ligne de patch actuelle
- les équipes peuvent conserver le chemin simple des aliases first-party sans contournement spécifique à Gemini
- la base publique actuelle couvre désormais à la fois la compatibilité des layouts auteur et le correctif repo-path Gemini

## Valeurs recommandées après v1.1.2

- gardez les aliases first-party comme chemin public normal d'installation lorsque le catalogue les fournit déjà
- gardez les installations gérées repo-path sur la ligne de patch actuelle ou plus récente
- si Gemini échouait auparavant, relancez le flux complet d'ajout multi-target sur `v1.1.2` ou plus récent

## Que faire maintenant

- passez à `v1.1.2` ou plus récent si les installations multi-target échouaient pendant l'étape Gemini
- relancez le flux d'ajout pour les aliases first-party concernés
- traitez `v1.1.2` comme la base publique actuelle pour les guides d'installation et les exemples
