# v1.1.0

Date de sortie : `2026-04-18`

## Pourquoi cette version est importante

`v1.1.0` est une version de durcissement pour les équipes qui traitent le dépôt comme propriétaire de l'état d'intégration des agents. Elle ne modifie pas le chemin recommandé par défaut, mais rend les flux d'ajout, de mise à jour et de suppression plus fiables dans la configuration multi-agent prise en charge.

## Ce qui a changé pour les utilisateurs

- les flux de cycle de vie des intégrations détenues par le dépôt disposent désormais d'une vérification end-to-end plus solide sur les chemins gérés Claude, Codex, Cursor et OpenCode
- la surface de version publique pointe désormais vers une référence `v1.1.0` cohérente dans les docs, les starters et les pins des paquets d'assistance
- le smoke d'upgrade du package d'exécution vérifie désormais la promotion vers la version publique actuelle du package au lieu de s'arrêter sur l'ancienne ligne de patch

## Ce que cela signifie en pratique

- si votre dépôt gère la configuration des agents pour plusieurs cibles, les opérations d'ajout, de mise à jour et de suppression sont désormais exercées de manière plus réaliste avant la publication
- si votre équipe suit les docs publiques ou les README des starters, les versions visibles correspondent désormais à la référence publiée actuelle au lieu de mélanger d'anciennes valeurs
- si vous publiez ou transmettez des dépôts basés sur des starters, l'histoire de la version est plus facile à expliquer car la référence, les docs et les pins des packages sont désormais alignés

## Valeurs par défaut recommandées après v1.1.0

- gardez Go comme valeur par défaut lorsque vous souhaitez la trajectoire de production générale la plus solide
- utilisez les chemins Node ou Python runtime-package partagés lorsque ces lanes interprétées repo-local sont intentionnelles pour l'équipe
- traitez les intégrations gérées détenues par le dépôt comme un cycle de vie validé lorsque le dépôt possède la configuration de l'agent

## Que faire maintenant

- mettez à jour toutes les docs copiées ou les extraits de starter qui mentionnent encore `v1.0.6`
- si vous dépendez d'une configuration d'intégration détenue par le dépôt entre agents, relancez votre flux normal d'ajout, de mise à jour et de suppression avec `v1.1.0`
- si vous repartez de zéro, utilisez les pins de starter actuels et cette note de version comme référence
