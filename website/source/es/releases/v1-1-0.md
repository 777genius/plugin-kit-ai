# v1.1.0

Fecha de lanzamiento: `2026-04-18`

## Por qué es importante esta versión

`v1.1.0` es una versión de endurecimiento para equipos que tratan el repositorio como el propietario del estado de integración del agente. No cambia la ruta recomendada por defecto, pero hace que los flujos de agregar, actualizar y eliminar sean más confiables en la configuración multiagente admitida.

## Qué cambió para los usuarios

- los flujos del ciclo de vida de integraciones propiedad del repositorio ahora tienen una verificación end-to-end más sólida en las rutas gestionadas de Claude, Codex, Cursor y OpenCode
- la superficie pública de la versión ahora apunta a una línea base consistente `v1.1.0` en docs, starters y pins de paquetes auxiliares
- el smoke de actualización del paquete de tiempo de ejecución ahora comprueba la promoción a la versión pública actual del paquete en lugar de detenerse en la línea de parche anterior

## Qué significa esto en la práctica

- si su repositorio administra la configuración del agente para más de un objetivo, las operaciones de agregar, actualizar y eliminar ahora se ejercitan de forma más realista antes del lanzamiento
- si su equipo sigue las docs públicas o los README de los starters, los pins de versión visibles ahora coinciden con la línea base publicada actual en lugar de mezclar valores antiguos
- si publica o entrega repositorios basados en starters, la historia del lanzamiento es más fácil de explicar porque la línea base, las docs y los pins del paquete ahora coinciden

## Valores predeterminados recomendados después de v1.1.0

- mantenga Go como predeterminado cuando desee la historia de producción general más sólida
- use las rutas compartidas de Node o Python runtime-package cuando esas líneas interpretadas repo-local sean intencionales para el equipo
- trate las integraciones gestionadas propiedad del repositorio como un ciclo de vida validado cuando el repositorio posea la configuración del agente

## Qué hacer ahora

- actualice cualquier doc copiada o fragmento de starter que aún mencione `v1.0.6`
- si depende de la configuración de integración propiedad del repositorio entre agentes, vuelva a ejecutar su flujo normal de agregar, actualizar y eliminar contra `v1.1.0`
- si comienza desde cero, use los pins actuales de los starters y esta nota de versión como línea base
