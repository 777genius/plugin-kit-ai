- keeping team discussions grounded in published guidance rather than outdated repo folklore

## Scope

- Public release notes.
- Migration notes.
- Breaking change callouts.

## What This Section Is Not

- It is not the best first place to learn the product from scratch.
- It does not replace guides for setup, target choice, or runtime choice.
- It is the right place after you already know your path and want to see what changed for users.

## Suggested Reading Order

1. Read the newest note that matches your runtime or delivery path.
2. Follow the "What this means in practice" and migration callouts there.
3. Go back to the matching guide or reference page only after you know whether the recommendation changed.
